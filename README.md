> AI 技能（Skill）—— 装在你的 AI 助手里，给它加一个能力：**提示词优化**。

## 怎么用

装好之后，跟你的 AI 助手说：

```
优化这个提示词：你是个设计师，帮我画个 logo
```

它会先告诉你缺了什么（诊断），然后给出三个版本，最后说明改了什么、为什么改。

支持的说法：`帮我优化提示词` / `改写提示词` / `improve my prompt` / `make this prompt better`。也支持直接把需求丢给它，不加前缀。

---

## 三个版本

| 版本 | 长度 | 场景 | 说明 |
|------|------|------|------|
| **极简版** | 1-3 句 | 自己用、快速搞定 | 角色到位、任务清晰，复制粘贴就能跑 |
| **规范版** | ≤10 句 | 日常工作、发给同事 | 加了上下文和关键约束，完整但不冗余 |
| **发布级版** | ≤20 句 | 高价值任务、写进文档 | 上下文丰富、质量标准明确、有负向约束 |

每个版本相互独立，可直接使用，不会出现"在上一版本基础上追加以下内容"。

## 效果

**输入**

```
你作为玻璃面板高级工程师，这是一份报告，请你深度分析里面存在的问题，技术性问题
```

**诊断**

> 角色：有领域、缺经验定位 / 约束：无边界 / 推理引导：无

**极简版**

> 你是玻璃面板高级研发工程师，专精质量分析与工艺缺陷诊断。请审查以下技术报告，从数据可靠性、工艺合规性、技术逻辑严密性三个角度分析问题并标注严重程度（高/中/低）。

**规范版**

> 你是玻璃面板行业的高级研发工程师，拥有 10 年以上产品质量分析与工艺缺陷诊断经验。以下是一份技术报告，请以资深评审者视角进行深度技术审查。
>
> 审查重点：数据与结论是否有充分对应关系；工艺参数和判定结果是否与相关标准一致；技术逻辑是否严密——因果推断是否排除了合理替代解释。对每个问题说明问题所在、为什么是问题、严重程度（高/中/低），按严重程度降序排列。只列有实质技术依据的问题，不凑数。

**发布级版**

> 你是玻璃面板行业的高级研发工程师，拥有 10 年以上产品质量分析与工艺缺陷诊断经验，熟悉强制性标准测试流程和常见失效模式。以下是一份技术报告，请以资深评审者视角进行深度技术审查。
>
> 审查重点：数据与结论是否有充分对应关系；工艺参数和判定结果是否与相关标准一致；技术逻辑是否严密——因果推断是否排除了合理替代解释，是否存在循环论证或跳跃性结论。
>
> 对每个问题说明问题所在、为什么是问题、严重程度（高/中/低），按严重程度降序排列。不凑数——没有实质技术依据的问题不列。报告整体质量高时，先肯定做得好的部分，再指出可改进之处。

## 原理

### 拿到输入后做什么

三步：诊断 → 重构 → 输出。

**诊断**：逐项检查 7 个维度，看你的原始提示词缺了什么。

| 维度 | 检查什么 | 常见问题 |
|------|---------|---------|
| 清晰度 | 目标明确吗？ | "帮我写个东西"——什么东西？ |
| 上下文 | 有背景信息吗？ | 没说受众是谁、什么场景 |
| 约束 | 有边界吗？ | 没说格式、长度、语调、范围 |
| 角色 | 指定了专家身份吗？ | "你是个设计师"——什么领域？什么级别？ |
| 结构 | 输出格式明确吗？ | AI 不知道你想要表格还是文章 |
| 示例 | 有参考样本吗？ | 没有 Few-shot 引导风格 |
| 推理引导 | 有思考方向吗？ | 复杂任务缺少分析框架 |

只列出真正有缺陷的维度。没问题的不硬凑。

**重构**：根据诊断结果补全缺失维度，分三层：

| 层级 | 骨架 | 适合 |
|------|------|------|
| Tier 1 | 角色 + 任务 + 格式 | 简单问答、一次性任务 |
| Tier 2 | + 上下文 + 约束 | 分析、创作、多步骤任务 |
| Tier 3 | + 示例 + 推理引导 | 复杂工作流、高价值输出 |

**输出**：三个版本，你挑一个。

### 为什么和其他工具不一样

大多数提示词优化工具的做法：给你塞模板、填表格、加一堆"请按以下格式输出"。

**这个技能反其道而行之——克制优先。**

| 做 | 不做 |
|----|------|
| 给 3-5 条核心原则 | 塞 15 条 checklist |
| "根据内容自行组织" | "按以下格式输出" |
| 定义 WHAT（审什么） | 规定 HOW（怎么想） |
| 信任模型判断力 | 规定死板的章节顺序和字数 |
| 10 行说清楚 | 80 行事无巨细 |

**大模型不是表格填写器。** 它比你想象中聪明，给它原则就够了，别用模板把判断力读傻了。

另外：

- ❌ 不规定思考流程（"先做A再做B最后做C"）
- ❌ 不为凑字数而添加冗余
- ❌ 不改变原始意图
- ❌ 不留占位符（`[insert here]`、`TBD`）

## 安装

### 前提条件

需要一个支持 Skill 机制的 AI 助手。目前支持：
- **WorkBuddy** — 原生支持，将技能文件放入 `~/.workbuddy/skills/` 目录即可
- **Claude Code** — 原生支持 `claude skills install` 命令
- **其他 AI 助手** — 将 SKILL.md 内容粘贴到 system prompt 中作为手动版技能

### WorkBuddy

**第一步**：克隆仓库到本地技能目录。

```bash
git clone https://github.com/kkkkkkk123216/prompt-optimizer.git ~/.workbuddy/skills/prompt-optimizer
```

> 也可以手动创建 `~/.workbuddy/skills/prompt-optimizer/` 目录，把仓库里的 `SKILL.md` 和 `references/` 复制进去。

**第二步**：重启 WorkBuddy 或刷新技能列表，技能会自动加载。

**第三步**：验证是否安装成功——输入 `优化这个提示词：帮我写个周报`，如果 AI 返回三版本优化结果，说明装好了。

最终目录结构：

```
~/.workbuddy/skills/prompt-optimizer/
├── SKILL.md              # 技能定义（触发条件、工作流、质量标准）
└── references/
    └── techniques.md      # 技术参考库（诊断维度、结构模式、反模式）
```

### Claude Code

```bash
claude skills install https://github.com/kkkkkkk123216/prompt-optimizer.git
```

安装完成后，在对话中输入 `优化这个提示词：<你的需求>` 即可触发。

### 手动安装（任意 AI 助手）

```bash
git clone https://github.com/kkkkkkk123216/prompt-optimizer.git
```

将 `SKILL.md` 的完整内容作为 system prompt / custom instructions 粘贴到你的 AI 助手中。之后在对话中说"优化这个提示词"即可触发优化流程。

### 常见问题

**Q: 安装后不生效？**  
A: 检查文件路径是否正确。WorkBuddy 要求技能目录在 `~/.workbuddy/skills/` 下，目录名即技能名。

**Q: 提示词太模糊，优化结果也一般？**  
A: 这是预期行为。技能会在模糊输入时反问一句补充信息，然后再优化。给它更多上下文，输出会更好。

**Q: 能改成只输出一个版本吗？**  
A: 目前固定三版本输出。如果你只需要某个版本，直接复制那一版就行，其他两个忽略即可。
