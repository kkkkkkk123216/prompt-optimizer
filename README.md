## 它是什么

一个 AI 编码助手技能（Skill），安装后你的 AI 助手会自动具备**提示词工程能力**——把模糊、随意的人类语言转化为结构严谨、可精确执行的专业提示词。

不是简单润色。是诊断、补全、重构。

## 它怎么工作

```
你输入的原始提示词
        │
        ▼
  ┌─────────────┐
  │  7 维诊断    │  ← 角色 / 上下文 / 约束 / 格式 / 示例 / 推理引导 / 清晰度
  └─────┬───────┘
        │
        ▼
  ┌─────────────┐
  │  智能重构    │  ← 按复杂度选择骨架/标准/高级三层结构，自动填补空白
  └─────┬───────┘
        │
        ▼
  ┌─────────────┐
  │  克制过滤    │  ← 消除过度约束、死板模板、扼杀自主组织等反模式
  └─────┬───────┘
        │
        ▼
  生产级提示词（可直接复制使用）
```

## 看看效果

**输入**：`帮我写个产品介绍`

**输出**：

> **缺失诊断**
> - 角色：未指定
> - 上下文：缺少背景信息和目标受众
> - 约束：未设定边界和质量标准
> - 输出格式：未指定
>
> **优化后提示词**
> ```
> 你是一名资深科技产品文案专家，拥有10年SaaS产品营销经验。
> 请为新上线的AI代码助手工具撰写产品介绍。目标受众是中小型软件公司的技术负责人。
> 要求：1) 突出核心功能 2) 强调ROI 3) 使用专业但易懂的技术语言 4) 字数控制在500字以内。
> 输出结构由你根据内容自行组织，怎么讲清楚就怎么讲。
> ```
>
> **改进说明**
> - 角色具体化：从"写手"升级为"资深科技产品文案专家 + 领域经验"
> - 上下文补全：明确产品类型、目标受众
> - 约束精炼：4条核心原则替代模糊要求
> - 输出信任：用"自行组织"替代死板模板，保留灵活性

## 核心设计理念

| 原则 | 做什么 | 不做什么 |
|------|--------|----------|
| **诊断而非批评** | 指出缺失维度 | 贬低原始输入 |
| **补全而非重写** | 填补空白 | 改变用户意图 |
| **克制而非过度** | 给 3-5 条核心原则 | 塞 15 条 checklist |
| **信任而非控制** | "根据内容自行组织" | 规定死板的章节顺序和字数 |

## 技术细节

### 7 维诊断框架

每个原始提示词都会在以下 7 个维度上被评估：

| 维度 | 检查什么 | 危险信号 |
|------|----------|----------|
| 清晰度 | 目标是否明确？ | "帮我改好" "写得好一点" |
| 上下文 | 是否有背景信息？ | 缺少受众、领域、场景 |
| 约束 | 边界是否清晰？ | 无格式、长度、语调限制 |
| 角色 | 是否指定了专家角色？ | 没有 persona 定义 |
| 结构 | 输出格式是否指定？ | 无章节、步骤、模板指导 |
| 示例 | 是否有少样本示例？ | 未演示期望的风格/质量 |
| 推理 | 是否引导了思考过程？ | 无 chain-of-thought 指令 |

### 三层重构策略

- **Tier 1 骨架**：角色 + 任务 + 输出格式（简单问答/一次性任务）
- **Tier 2 标准**：+ 上下文 + 约束（分析/创作/多步骤任务）
- **Tier 3 高级**：+ 示例 + 推理引导（复杂工作流/高重要性输出）
